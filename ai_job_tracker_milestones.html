M1
Foundation & Auth
3–4 ngày · Spring Security + JWT + Next.js Login
▼
Backend
  • Khởi tạo Spring Boot project: Web, JPA, Security, PostgreSQL, Lombok, Validation
  • Entity User: id (UUID), email, passwordHash, createdAt
  • JWT filter chain: UsernamePasswordAuthenticationFilter → issue token, OncePerRequestFilter → validate
  • Endpoints: POST /auth/register, POST /auth/login, GET /auth/me (protected)
  • Docker Compose: PostgreSQL 16 + pgAdmin (optional)
  • Lưu ý: hash password bằng BCrypt, không plain text
Frontend
  • Setup Next.js 14 + TypeScript + Tailwind + shadcn/ui
  • Trang /login và /register — gọi API, lưu JWT vào httpOnly cookie hoặc localStorage
  • Middleware Next.js: redirect nếu chưa login
  • Axios instance với interceptor tự gắn Authorization: Bearer ...
✅ Done khi: đăng ký → đăng nhập → nhận JWT → gọi /auth/me trả về thông tin user
Spring Security JWT (jjwt) PostgreSQL Next.js 14 Docker Compose
M2
Job CRUD + Kanban Board
4–5 ngày · Kéo thả pipeline, data persist
▼
Backend
  • Entity Job: id, userId (FK), title, company, logoUrl, status, url, notes, deadline, matchScore, createdAt, updatedAt
  • Enum JobStatus: SAVED → APPLIED → INTERVIEW → OFFER → REJECTED
  • CRUD endpoints: POST /jobs, GET /jobs, PUT /jobs/{id}, DELETE /jobs/{id}
  • Endpoint riêng: PATCH /jobs/{id}/status — dùng cho kéo thả Kanban
  • Validation: deadline không được ở quá khứ, title không rỗng
Frontend
  • Kanban board dùng @dnd-kit/core — 5 cột tương ứng 5 status
  • Job card hiện: logo công ty (Clearbit API free), tên vị trí, match%, deadline badge đổi màu khi gần
  • Drag end → gọi PATCH /jobs/{id}/status → optimistic update UI
  • Modal tạo/sửa job với form validation (react-hook-form + zod)
✅ Done khi: tạo job → kéo sang cột khác → refresh trang → vẫn đúng cột
@dnd-kit/core react-hook-form zod Clearbit API JPA / Hibernate
M3
Scraper + AI Analyzer
5–6 ngày · Phần core nhất của project
▼
CV Upload (thêm vào M1 hoặc làm đầu M3)
  • Entity CV: id, userId (FK), fileName, rawText, uploadedAt, isActive
  • Dùng Apache PDFBox parse PDF → extract text → lưu vào cv.rawText
  • Lưu file gốc: MinIO (self-hosted, free) hoặc đơn giản hơn là lưu thẳng DB dạng BYTEA
  • Endpoint: POST /cv/upload (multipart), GET /cv/active
  • User chỉ có 1 CV active tại một thời điểm — logic: upload mới thì set isActive cũ = false
Scraper
  • Dùng Jsoup fetch HTML từ link JD
  • Ưu tiên ITviec trước (HTML ổn định, dễ parse). Selector ví dụ: .job-details__content
  • Fallback: nếu scrape thất bại, user tự điền tay vào form
  • Trả về: title, company, description, requirements (dạng text thuần)
  • Cẩn thận: LinkedIn block scraping — bỏ qua hoặc dùng paste thủ công
AI Service
  • Gọi OpenAI GPT-4o mini (rẻ, đủ dùng) hoặc Gemini Flash (có free tier)
  • Prompt template: so sánh cv.rawText với JD, trả về JSON: matchScore, missingSkills[], suggestedKeywords[], summary
  • Cache kết quả vào Redis với key = hash(cvText + jdText), TTL 24h — tránh gọi API lại
  • Lưu kết quả vào job.matchScore + bảng ai_analysis
✅ Done khi: paste link JD → scrape → AI chạy → hiện match%, skills thiếu trong job detail panel
Jsoup Apache PDFBox OpenAI API Redis Spring Cache
M4
Email Reminder (không có Kafka)
2–3 ngày · @Scheduled + JavaMailSender
▼
Tại sao không cần Kafka?
  • Monolith = 1 process → không cần message broker để giao tiếp giữa services
  • @Scheduled chạy trong cùng JVM, đủ mạnh cho use case này
  • Kafka chỉ cần khi: nhiều microservice, cần replay event, traffic cực cao
Implementation
  • Thêm @EnableScheduling vào main class
  • Class ReminderScheduler với 2 jobs:
  •   → Chạy mỗi ngày 8h sáng: query job có deadline < now + 24h AND status != REJECTED → gửi email
  •   → Chạy mỗi ngày 9h sáng: query job có status = APPLIED AND updatedAt < now - 14 days → gửi email nhắc follow-up
  • Gmail SMTP free: spring.mail.host=smtp.gmail.com, port 587, bật App Password
  • Email template HTML đơn giản (Thymeleaf hoặc string thuần)
  • Thêm cột job.lastReminderSentAt để tránh gửi email trùng lặp
✅ Done khi: job gần deadline → nhận email nhắc đúng giờ, không bị gửi lặp
@Scheduled JavaMailSender Gmail SMTP Thymeleaf
M5
Analytics Dashboard
3–4 ngày · Recharts + aggregate queries
▼
Backend — Analytics API
  • GET /analytics/summary: tổng job, tỉ lệ offer/applied, avg match score
  • GET /analytics/funnel: số lượng job ở mỗi stage (cho funnel chart)
  • GET /analytics/weekly: jobs applied theo từng tuần (7 tuần gần nhất)
  • GET /analytics/skills: top 10 skills hay thiếu nhất (aggregate từ bảng ai_analysis)
  • Dùng JPQL aggregate query thay vì tính ở Java → nhanh hơn
Frontend
  • Funnel chart pipeline (Saved → Offer) — dùng FunnelChart của Recharts hoặc tự vẽ SVG
  • Line chart: jobs apply theo tuần — LineChart Recharts
  • Bar chart ngang: top skills thiếu — BarChart horizontal
  • Summary cards: tổng apply, response rate, avg match score
✅ Done khi: dashboard hiện số liệu thực từ jobs đã tạo, charts update khi thêm job mới
Recharts JPQL @Query
M6
Docker + Deploy
2–3 ngày · 1 lệnh chạy được, có link public
▼
Docker Compose (local + prod)
  • Services: backend (Spring Boot JAR), frontend (Next.js), postgres, redis
  • Không có Kafka → file docker-compose.yml gọn hơn nhiều
  • Dùng .env file cho secrets: DB password, JWT secret, OpenAI key, Gmail password
  • Health check cho postgres trước khi backend start
  • Multi-stage Dockerfile cho backend: build JAR ở stage 1, chạy ở stage 2 (image nhỏ hơn)
Deploy (chọn 1)
  • Railway (khuyến nghị): free tier ổn, deploy bằng git push, có managed PostgreSQL và Redis
  • Render: tương tự Railway, free tier có spin-down sau 15 phút idle
  • VPS (rẻ nhất): Vultr/DigitalOcean $4/tháng, tự cài Docker, full control

  • Demo GIF/video ngắn 30s ở đầu README
  • Tech stack badges, architecture diagram
  • Hướng dẫn chạy local: clone → cp .env.example .env → docker compose up
  • Link demo public + tài khoản test sẵn
✅ Done khi: docker compose up chạy được, có link public hoạt động
